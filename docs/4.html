实验工具指南

  1. (gdb) info threads：列出所有的线程
  2. (gdb) thread 2：切换到线程2执行
  3. (gdb) set print pretty：启用数组和结构的美化输出
  4. (qemu) info registers：列出全部寄存器，gdb只能看通用寄存器和段寄存器
  5. (qemu) info mem：映射的虚拟内存和权限（ef7c0000-ef800000 00040000 urw 表示0xef7c0000 到 0xef800000 长度0x40000字节的内存权限，u用户态，r读，w写）

实验1

  1. 对比《x86架构操作系统内核的实现》，启动分页的方式很类似
  2. obj/kern/kernel.img的生成脚本在：kern/Makefrag
    • obj/kern/kernel.img第一个扇区放：obj/boot/boot
      • 源文件：boot/boot.S，开启A20，开启保护模式，gdtr指向gdt，esp=0x7c00
      • 源文件：boot/main.c，从磁盘读取内核文件到0x10000处，解析elf，跳转到ELFHDR->e_entry（kern/kernel.S）
    • obj/kern/kernel.img第二个扇区起：obj/kern/kernel
  3. 启动分页前（现状）
    • 内核文件的虚拟地址都是从0xF000000起
  4. 启动分页前（kern/entrypgdir.c，定义页目录和页表）
    • 虚拟0~4M，对应物理0~4M
    • 虚拟3.75G开始的0~4M，仍然对应物理0~4M
  5. 启动分页（kern/entry.S）
    • 此文件是内核的入口执行位置，由boot/main.c读取elf跳转而来
    • 由于虚拟地址都是从0xF000000起，未启动分页，所以地址都需要减0xF000000
    • 此文件的作用就是设置kern/entrypgdir.c定义的页目录和页表，开启分页，esp=bootstacktop
    • 最后跳转到kern/init.c执行i386_init()

实验2 物理内存

  1. 全局变量edata：bbs段起始位置
  2. 全局变量end：bbs段结束位置
  3. 全局变量npages：内存的总页数
  4. 全局变量npages_basemem：已用内存的页数
  5. void * boot_alloc(uint32 n)：物理内存分配器（仅用于虚拟内存初始化前），真正的物理内存分配器是page_alloc
  6. mem_init()：
    • kern_pgdir：页目录表，分配1页
    • pages：物理页表，分配npages页
    • page_init()：物理页用单链表实现了后进先出的栈，每次分配物理页即是出栈
  7. PageInfo *page_alloc(int alloc_flags)：出栈（获取物理地址还需要调用page2pa）
  8. page_free(PageInfo *pp)：入栈，page_free_list指向栈顶

实验2 虚拟内存

  1. 全局常量UPAGES：User Pages，让用户态也能访问物理页表
  2. 内核栈大小8个物理页，即32K
  3. pgdir_walk()：通过va来查页表项，页表项不存在则创建
  4. boot_map_region()：内存映射，从va处申请n字节内存
  5. page_lookup()：通过va找到pa
  6. page_remove()：取消虚拟地址与物理地址的关联
  7. page_insert()：
  8. 三次boot_map_region()：
    • 0xef000000~0xef400000，对应物理地址：0x11b000~0x51b000，共4M，存物理页表
    • 0xefff8000~0xf0000000，对应物理地址：0x10e000~0x117000，共32K，存内核栈
    • 0xf0000000~0xffffffff，对应物理地址：0x0000~0x10000000，共256M
