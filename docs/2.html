CPU知识

  1. x86/x64/x86_64/i386/ia32/ia64/amd/amd64 辨析
  2. 四大主流芯片架构（X86、ARM、RISC-V和MIPS）
  3. X86并不是指令集，而是一套向后兼容指令集架构，它包含了16位指令集，32位指令集、64位指令集
    • x64 = x86_64 = amd64
      1. 是对IA-32的扩展，由AMD提出，Implemented by AMD、Intel。可兼容32位指令集(IA-32)
      2. 为什么叫x86-64，因为它是X86的一部分，兼容X86的其他指令集(32-bit、16-bit)
    • IA-32 = i386，32位的X86指令集
    • IA-64，Intel提出的64位指令集，目前比较鸡肋
  4. 四大架构：X86、ARM、RISC-V、MIPS
    1. X86：CISC 复杂指令集，英特尔和AMD的专属，在PC市场上独霸多年
    2. ARM：RISC 精简指令集，既是公司名，也可以指该公司的架构，在移动端和便捷设备上有着不可替代的优势
    3. RISC-V：基于RISC的开源架构，在智能穿戴产品上的应用广泛，前景广阔
    4. MIPS：英国公司的指令集，在网关、机顶盒等市场上非常受欢迎
  5. CPU芯片，中国的芯片现状如何？-知乎
    • 芯片三大领域：设计、制造、封装
      1. 设计：海思、高通、AMD、展讯、联发科
      2. 制造：主流台积电、三星、中芯国际、Intel
      3. 封装：长电科技，台湾日月光
    • 相关产业：内存方面是三星、东芝、镁光、长江存储等；屏幕方面是三星、LG、夏普、京东方、天马微等
    • 生产资料：晶圆主要是日本厂商信越半导体和胜高科技；光刻机主要是荷兰的ASML；EDA厂商主要是Synopsys、Cadence、Mentor三家
    • 在芯片三大领域中，附加值较低的封装国内已经达到全球先进水平
    • 国内芯片行业目前是：封装>内存和屏幕>芯片设计>芯片制造>晶圆光刻机和EDA
    • 中国芯片到底怎么样？
    • 现在有哪些中国大陆产 CPU 和操作系统？现状如何？
    • 国内芯片设计最先进的是华为海思(手机嵌入式)、紫光展讯(5G通讯)、龙芯中科(PC服务器)、阿里平头哥(服务器)、长江存储(NAND闪存)
    • 国内芯片设计分为自主设计和买授权：
      1. 龙芯中科：从下到上都是自主设计，指令集是模仿的
      2. 买授权自己设计Soc：成都海光（x86）、上海兆芯（x86）、飞腾（ARM）、海思（ARM）、展讯（ARM）等
      3. 史上最全国产服务器CPU芯片大起底
    • 国内芯片制造最先进的是中芯国际和厦门联芯，目前能做到28纳米量产。三星、台积电等巨头即将在今年量产7纳米，相差两三代。
    • 国内芯片制造：
      1. 高端产品：CPU、内存、Flash存储等，难度大。除了收购别人已经没有捷径可言，可是稍微好点的又限制中国收购（镁光）
      2. 低端产品：可以制造。
      3. 工具：光刻机虽然是别人不卖，但主要原因还是工业积累的时间不够长
      4. 制造芯片比设计芯片更烧钱
    • 美国芯片的优势：
      1. 芯片底层架构授权，如X86、ARM以及大量IP和专利。
      2. 大量的芯片设计，如没有高通芯片，中兴手机则没有旗舰级芯片。
      3. EDA工具，若无EDA公司支持，芯片公司无法进行研发。
      4. 产业链和资本控制，虽然日本、韩国、台湾的芯片行业都很强，但都深受美国影响和控制。

系统调用

  • 操作系统学习（十四） 、中断描述符表和中断描述符
  • 电脑开机后是实模式，此时可以访问任何内存，通过指令变成保护模式
  • GDT表：Global Descriptor Table，全局描述符表。item称为段描述符。
    • 段描述符：8字节，主要包含：段的基址 + 段的总页数 + 读写权限
  • LDT表：Local Descriptor Table，局部描述表，每个任务可以有一张。item称为段描述符。GDT和LDT的区别
  • IDT表：Interrupt Descriptor Table，中断描述符表。可以存放三种类型的门描述符：
    1. 中断门描述符：16位segment selector，32位offset
    2. 陷阱门描述符：16位segment selector，32位offset
    3. 任务门描述符
    4. 最多256个item，每个item固定8字节，寻址 = IDTR + item的索引 * 8，其中的DPL字段标识了特权级别
  • TSS段：Task Status Segment，linux的TSS段只有1个item。浅谈tss
    1. 有一个TR寄存器指向该item，且永远指向它
    2. linux的TSS段中只使用esp0和iomap等字段，当从用户态切到内核态时，esp0指向内核栈的地址，其它的寄存器则在esp0指示的内核栈
    3. 进程切换时，只更新唯TSS段中的esp0字段，esp0保存新进程的内核栈地址。
  • 寄存器：
    1. CPL：Current Privilege Level，当前的特权级别（0~3）
    2. GDTR：指向GDT表的位置
    3. LDTR：指向LDT表的位置
    4. IDTR：指向IDT表的位置，48位（其中32位表示LDT表基地址，16位表示LDT表的长度）
    5. 6个段寄存器：保护模式 2讲 -段 -段寄存器结构
      • CS(代码段)
      • SS(堆栈段)
      • DS(数据段)
      • ES(附加扩展段)
      • FS(分段机制)
      • GS(备用)
      • 段寄存器16位：13位是segment selector，1位是TI（Table Indicator），2位是RPL（Requested Privilege Level）
        • TI=0，找GDT表
        • TI=1，找LDT表
      • Intel为了保持向后兼容，将段寄存器仍然规定为16位，所以需要借助GDT/LDT表找到基址
  • 汇编指令：
    1. 开启A20：
      mov dx,0x92
      in al,dx
      or al,0x02
      out dx,al
    2. 开启保护模式：
      cli
      mov eax,cr0
      or eax,0x01
      mov cr0,eax
    3. 从实模式到保护模式：CR0寄存器PE位置为1
    4. LIDT 和 SIDT 分别用于加载和保存IDTR寄存器，CPL=0才能执行
  • 通过int 80指令从用户态到内核态的新方案：使用sysenter和和sysexit指令
  • linux内核栈：怎么理解linux内核栈？
    • 一，内核线程(如idle，kthread)使用的栈，这些仅在内核空间运行，只有内核栈，没有用户态
    • 二，用户态切到内核态时使用的栈，一般4k或8k，每个进程对应一个此栈
    • 三，中断处理的栈
    • 系统调用最初的栈是共用的，在cpu_tss那里，只有256字节，进入核心之后，再把堆栈切换到当前线程的栈顶
    • 中断处理的栈所以进程共享，但每个虚拟cpu核心有一份。
    • 内核态没有进程堆的概念，用kmalloc()分配内存，一般用slab分配器
